37 /* Now we define a test driver that runs tests in the machine given by parameter
38 
39  Note that the driver waits for a run command (eg from iosh: SEND driver.run)
40 
41  The test machine is expected to enter a working state while its run
42  command executes and then to enter an error or ok state when the
43  tests are completed.
44 
45  To reset the test script, it is disabled and reenabled; this process
46  causes a machine to reinitialise.
47 
48  If the machine under test does not change state within 10 seconds (10000ms),
49  it is assumed to have stalled and the driver regards this as an error.
50 */
51 Driver MACHINE test {
52  script TestScript test;
53  error WHEN SELF IS error || SELF IS waiting && TIMER > 10000;
54  ok WHEN script IS ok ;
55  waiting WHEN script IS working;
56  idle DEFAULT;
57 
58  COMMAND run { SEND run TO script; WAITFOR script IS working }
59  COMMAND abort { DISABLE script; ENABLE script; }
60  ENTER error { LOG "error"; CALL abort ON SELF }
61 }

64 /* Here is the TestScript; during the working state, the machine
65  changes the values of the inputs to the test and monitors
66  the system output.
67 
68  The test we are using is the above ComparisonTest machine and
69  this machine should enter the state of lt, gt or eq by comparing
70  its inputs. If the machine moves to an error state, the
71  script stalls and the driver will soon notice this and raise
72  the error.

74  For convenience, the test machine keeps a step property as a
75  trace variable to indicate to the driver which step has failed.
76 */
